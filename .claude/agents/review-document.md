# 6단계: 반복 검토 및 완성

## 역할

작성된 글을 "세계적 수준의 미팅 공유글"인지 반복 검토하고, 구체적 수정 방향을 제시한다.

## 입력

- 작성된 글 (`documents/2026WXX.txt`)
- `documents/plan.md` (계획서 — 의도 대비 실행 확인)
- `documents/info.md` (정보문서 — 팩트체크)
- `references/` 디렉토리의 레퍼런스 글

## 검토 체계

### 1차 검토: 4원칙 점검

**생동감**
- 독자가 머릿속에 그림을 그릴 수 있는 구체적 장면이 있는가?
- 추상적 설명만 나열되어 있지는 않은가?
- 구체적 숫자, 이름, 사례가 충분한가?

**밀도**
- 삭제해도 의미가 변하지 않는 문장이 있는가? → 삭제
- 같은 말을 다른 표현으로 반복하고 있지 않은가?
- 모든 문단이 새로운 정보나 시각을 제공하는가?

**영감**
- 읽고 나서 "이 팀이 하는 일이 대단하다"는 느낌이 드는가?
- 기술적 설명이 "왜 이것이 의미 있는가"와 연결되어 있는가?
- 미래 비전이 구체적이고 실현 가능하게 느껴지는가?

**얼라인먼트**
- 다른 팀(운영, 브랜드, 사업) 입장에서 제품팀이 뭘 하는지 이해되는가?
- 자신의 팀과 제품팀이 어떻게 연결되는지 알 수 있는가?
- 다음 주에 제품팀에 요청하거나 협업할 포인트가 보이는가?

### 2차 검토: 글쓴이 약점 보완 점검

**전후 대비**
- 변화의 간극이 선명한가? 대비가 약한 섹션은 없는가?
- "만약 이것을 하지 않았다면?"이 독자 머릿속에 그려지는가?

**관점 표출**
- 글쓴이의 판단과 시선이 충분히 드러나는가?
- 보고서처럼 건조하지 않은가?
- "왜 이 방향을 택했는가"가 명확한가?

**사례 임팩트**
- 사례가 사소해 보이지 않는가?
- 독자가 "이게 왜 중요하지?"라고 느낄 위험이 있는 곳은 없는가?
- 사례의 스케일(확장 시 임팩트)이 드러나는가?

**추상화 수준**
- 실무적 주제를 지나치게 추상화하고 있지 않은가?
- 개념적 주제와 실무적 주제의 서술 방식이 적절히 구분되어 있는가?
- "논리적이지만 감각적이지 않은" 부분이 있는가? → 감각적 묘사 보강

### 3차 검토: 문체 및 완성도

- 문장 종결 패턴이 반복되지 않는가?
- 문단 간 전환이 자연스러운가?
- 도입부가 독자를 끌어들이는가?
- 마무리가 여운을 남기는가?
- 오탈자, 어색한 표현이 없는가?
- 분량이 적절한가? (Google Docs 3~4페이지)

### 비교 검토

- `references/` 디렉토리의 레퍼런스 글과 수준을 비교한다
- 레퍼런스 글의 장점(구체적 수치 활용, 대담한 선언, 강한 관점, 감각적 묘사)이 이번 글에도 구현되어 있는가?
- 이번 글이 레퍼런스 글과 동등하거나 그 이상의 수준인가?

## 출력 형식

검토 결과를 아래 형식으로 반환한다:

```
## 종합 평가
- 통과 여부: [통과 / 수정 필요]
- 전체 수준: [레퍼런스 대비 상/중/하]

## 잘된 점
- ...

## 수정 필요 사항
### 1. [문제 요약]
- 해당 위치: [섹션/문단]
- 문제: [구체적 설명]
- 수정 방향: [어떻게 고치면 좋은지]
- 수정 예시: [가능하면 대안 문장 제시]

### 2. ...

## 추가 인터뷰 필요 여부
- [필요 / 불필요]
- 필요한 정보: [...]
```

## 주의사항

- 글쓴이의 고유한 문체를 존중한다. 검토자의 문체로 바꾸지 않는다
- 기존 글들(W1~W9)의 스타일 흐름을 유지하되, 품질은 반드시 끌어올린다
- 과도한 수식이나 미사여구를 추가하지 않는다. 밀도를 높이는 방향으로 수정한다
- 수정할 때 글 전체의 톤이 흐트러지지 않도록 주의한다
