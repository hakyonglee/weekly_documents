# 5단계: 글 작성

## 역할

정보문서와 글쓰기 계획서를 바탕으로 최종 글을 작성한다.

## 입력

- `documents/plan.md` (글쓰기 계획서)
- `documents/info.md` (정보문서)
- `documents/material.txt` (원본 재료 — 팩트체크용)

## 출력

- `documents/2026WXX.txt` (해당 주차 번호)

## 작성 원칙

### 톤과 문체

- 격식체를 사용하되 딱딱하지 않게. "말을 거는 격식체"
  - 좋은 예: "하지만 안타깝게도 AIOS는 1차원적인 답변을 내놓는 데 그치고 있었습니다."
  - 좋은 예: "우리는 '최선을 다하고 있다'는 추상적인 위안에서 벗어나야 합니다."
  - 나쁜 예: "AIOS의 답변 품질이 낮은 것으로 확인되었습니다."
- 능동태를 기본으로. 수동태는 최소화한다
- 문미 반복을 피한다. "~것입니다", "~있습니다"만 반복되지 않도록 다양한 종결어미를 사용한다

### 생동감

- 추상적 설명 전에 구체적 장면을 먼저 보여준다
- "~한 상황을 가정해 보겠습니다" 같은 도입으로 독자를 장면 안으로 끌어들인다
- 숫자, 이름, 시간, 장소 등 구체적 디테일을 넣는다
- 독자가 "아, 그런 상황이구나"라고 고개를 끄덕일 수 있어야 한다

### 밀도

- 모든 문장이 정보를 전달하거나 독자의 이해를 진전시켜야 한다
- "~하고자 합니다", "~라고 할 수 있습니다" 같은 불필요한 완충 표현을 줄인다
- 한 문단에 하나의 핵심 포인트만 담는다
- 문단이 5줄을 넘기면 나눌 수 있는지 검토한다

### 대비 — 약점 보완 핵심

- Before/After를 명시적으로 보여준다
- 대비의 간극이 클수록 독자의 인상이 강해진다
- 대비를 통해 "왜 이 일이 중요한가"를 느끼게 한다
- 단순히 "이전에는 A였는데 지금은 B입니다"가 아니라, A 상태의 고통과 B 상태의 가치를 각각 묘사한다

### 관점 — 약점 보완 핵심

- "제품팀은 ~라고 판단했습니다"처럼 주어를 명확히 한다
- 결정의 이유를 밝힌다. "왜냐하면~", "그 이유는~"
- 확신 있는 문장을 쓴다. "~인 것 같습니다" 같은 회피 표현을 피한다
- 대안을 검토한 과정을 보여주면 판단에 무게가 실린다

### 추상화 수준 조절 — 약점 보완 핵심

- 계획서에 명시된 각 섹션의 추상화 수준을 따른다
- 개념적 주제: 아키텍처, 원리, 전략을 논리적으로 설명
- 실무적 주제: 현장 장면, 사용자 반응, 구체적 변화를 감각적으로 묘사
- 모든 것을 동일한 수준으로 추상화하지 않는다. 이것이 가장 중요한 포인트다

### 구조

- 도입부(1~2문단): 이번 주 핵심 메시지를 압축적으로 전달
- 본문: 계획서의 목차를 따르되, 읽는 흐름이 자연스럽도록 연결
- 마무리(1문단): 다음 주 계획과 방향성
- 섹션 간 전환이 매끄러워야 한다. 갑작스러운 주제 전환 금지

### 분량

- Google Docs 기준 3~4페이지
- 텍스트 파일 기준 약 3,000~5,000자(공백 포함)
- 아젠다가 여러 개일 때도 전체 분량은 이 범위를 유지한다. 각 아젠다의 깊이를 조절한다
