# 3단계: 글쓰기 계획

## 역할

정보문서(`documents/info.md`)를 바탕으로 글의 구조, 목차, 각 섹션의 핵심 내용을 설계한다.

## 입력

- `documents/info.md` (정리된 정보문서)
- `documents/material.txt` (원본 재료)
- 기존 글들 (`documents/2026W*.txt`) — 스타일 및 흐름 참고
- `references/` 디렉토리 — 품질 벤치마크

## 출력

`documents/plan.md`

## 계획서 구조

```
# 글쓰기 계획서

## 이번 주 핵심 메시지
- 이 글 전체를 관통하는 하나의 메시지 (여러 아젠다를 관통하는 상위 맥락)

## 도입부 전략
- 어떻게 시작할 것인가
- 독자를 어떻게 끌어들일 것인가

## 목차 및 섹션별 계획

### 섹션 1: [제목]
- 핵심 내용 요약 (2~3문장)
- 사용할 사례/데이터
- 대비 구조: [Before] → [After]
- 글쓴이 관점이 드러나는 포인트
- 이 섹션의 추상화 수준: [개념적 → 추상적 설명 가능] 또는 [실무적 → 감각적 묘사 필요]

### 섹션 N: [제목]
...

## 마무리 전략
- 어떻게 끝낼 것인가
- 다음 주와의 연결

## 서사 설계 메모
- 글 전체의 흐름과 리듬
- 어디서 긴장감을 높이고, 어디서 해소하는가
- 어떤 부분에서 독자가 "이건 대단한데"라고 느끼게 할 것인가
```

## 계획 시 고려사항

### 구조 설계
- 아젠다가 여러 개일 때: 각 아젠다가 독립적으로도 읽히면서, 전체적으로 하나의 맥락으로 이어지는 구조를 설계한다
- 아젠다가 하나일 때: 문제-분석-해결, 현황-비전-실행 같은 서사 구조를 활용한다
- 각 섹션의 분량 배분을 미리 정한다
- 전체 3~4페이지 안에서 각 아젠다의 깊이를 조절한다

### 사례 선택
- 사례는 독자가 "이건 중요한 일이구나"라고 느끼게 해야 한다
- 사례가 사소해 보일 위험이 있으면:
  - 그 사례의 스케일을 보여준다 (이게 매일 반복되면? 9개 지점에서 동시에 일어나면?)
  - 더 임팩트 있는 사례로 교체한다
- 하나의 사례를 여러 섹션에 걸쳐 활용하면 관통 서사가 된다 (W7의 깐양파 사례처럼)

### 추상화 수준 판단 — 핵심
- 개념적/아키텍처 주제 (예: Memory 시스템 구조): 추상적으로 설명해도 좋다
- 실무적/현장 주제 (예: 입고 누락 방지, 품절 대응): 반드시 감각적으로 묘사한다. 현장에서 실제로 벌어지는 장면을 보여준다
- 계획서에 각 섹션의 추상화 수준을 명시한다

### 대비 설계
- 모든 핵심 주장에는 대비 구조가 필요하다
- "만약 ~하지 않았다면"(반사실적 대비)이 효과적
- 숫자의 대비, 상태의 대비, 관점의 대비를 섞어 사용한다
